3.3.1 时钟树介绍

对于STM32中，不同外设使用不同时钟频率，例如看门狗、RTC，只需要几十KHz就可以工作。针对不同部分进行详细说明。

针对STM32有：LSI、LSE、HSI、HSE、 PLL 时钟源。

  • ==高速时钟源==：HSI、HSE、 PLL。
  • 低速时钟源：LSI、LSE。
  • ==外部时钟源==：LSE、HSE。
时钟源 含义
LSI 内部低速时钟，32KHz，可供看门狗、RTC。
LSE 外部低速时钟，PC14、PC15，32.768KHz，供RTC使用。
HSI 内部高速时钟，RC振荡器，16MHz，可供系统时钟或PLL锁相环的输入。
HSE 外部时钟，4-26MHz，可供系统时钟和PLL锁相环输入，还可以经过2-31分频后输入给RTC。
PLL 锁相环，可将开发板的8MHz倍频，以达到芯片168MHz；
PLL分为主PLL和专用PLL；
主PLL是由HSI和HSE时钟输入，输出两个：PLLP、PLLQ；
PLLP：生成高速的==系统时钟==（最高168MHz），PLLQ：USB OTG FS（48MHz）、随机器时钟以及SDIO时钟。
专用 PLL(PLLI2S)：生成精确时钟，从而在 I2S 接口实现高品质音频性能。
==PLL=(HSE(HSI)/M)xN/P==。

至于各种外设时钟：

外设部分 说明
A 看门狗时钟，只能使用LSI时钟。
B RTC，实时时钟，能使用LSI、LSE、HSE分频。
C 时钟输出，MCO1：LES、HSE、HSI、PLL，MCO2：SYSCLK、PLL、HSE、PLL2S。（最大100MHz）
D ==系统时钟==，可用：HSI、HSE、PLLCLK，一般使用PLLCLK。
E AHB时钟，由SYSCLK提供，可分为：AHB、APB1 和 APB2；
AHB 8分频后给 Cortex 系统定时器Systick提供时钟；
AHB最大168MHz，APB2为 2 分频，最大84MHz，APB1为 4 分频，最大42MHz；
以太网PTP就是系统时钟。
F I2S时钟，由PLLI2SCLK、外部I2SCKIN提供。
G 内部以太网MAC时钟。
H USB OTG HS（24-60MHz）。

5. 实战篇

5.1 延时程序的编写

大家应该都使用过基于HAL库、标准库的延时程序，通常有以下几种方式：

  • 系统定时器Systick
  • HAL_Delay()函数
  • 指令延时

下面我要记录的是基于内嵌汇编指令延时程序设计：

针对汇编部分进行详细说明：

  • “1: subs %0, #1\n”

    • subs 是 ARM 汇编指令，表示将寄存器 %0 的值减去立即数 1，并更新条件标志（如零标志）。

    • subs 指令不仅会执行减法运算，还会影响条件标志位，这在 ARM 处理器中是非常常见的用法。

    • 1: 是一个标签，表示此位置的指令可以被引用，例如用于跳转。

    • %0 是 GCC 内嵌汇编的占位符，表示将使用输入操作数 cycles 对应的寄存器。

  • " bne 1b\n"

    • bne 是 ARM 汇编指令，表示“如果不等于零则跳转”（Branch if Not Equal）。

    • bne 指令会检查之前指令（这里是 subs 指令）的结果是否为零。如果结果不为零，则跳转到标签 1，即返回执行 subs 指令。

    • 1b 中的 1 表示跳转到标签 1，b 表示向后跳转（从当前指令位置往前找标签）。

  • : “=r” (cycles)

    • =：表示这是一个输出操作数，汇编代码会修改这个操作数，并将结果存储在一个寄存器中。
    • r：表示使用一个通用寄存器来存储这个操作数，编译器会选择一个适当的寄存器。
    • (cycles)：这是 C 变量 cycles，它在汇编代码执行后会被更新。

  • : “0” (cycles)

    • "0"：表示这个输入操作数使用与第一个输出操作数相同的寄存器，也就是说，重新赋值cycles变量，让其恢复初始值，不影响其它地方的使用。

    • (cycles)：这是 C 变量 cycles 的初始值，编译器会将这个值加载到前面指定的寄存器中。
